## Project Overview

PROVUS is an autonomous AI trading agent that solves the core problem of **verifiable AI decision-making in finance**. Every trading signal is generated by DeepSeek V3.1 running inside a **0G Compute Trusted Execution Environment (TEE)**, cryptographically signed, then permanently recorded on **0G Chain mainnet** — creating an immutable, time-stamped proof that each decision was made *before* execution, not fabricated after.

The system runs fully autonomously every **15 seconds**: fetch live market data → calculate volatility → query AI via TEE → attest on-chain → update ELO reputation → broadcast proof to frontend. No human intervention. No off-chain logs that can be tampered with. Everything verifiable on 0G ChainScan.

**60,000+ transactions** verified on 0G Chain mainnet. **30,000+ loop iterations** completed. **99.7% uptime.** Built for the 0G APAC Hackathon 2026.

**What it is:** EVM-compatible layer-1 blockchain (ChainID 16661) used as the tamper-proof state layer.

**How PROVUS uses it:** All 5 smart contracts are deployed on 0G Chain mainnet. ### ✅ Component 2: 0G Compute Network (TEE Inference)

**What it is:** Decentralized inference network with Trusted Execution Environment support. PROVUS uses DeepSeek V3.1 hosted by provider `0xd9966e13a6026Fcca4b13E7ff95c94DE268C471C` via the `@0glabs/0g-serving-broker` SDK.

**How PROVUS uses it:** Every 15 seconds, the agent encrypts a market context query (current price, volatility regime, trend direction) and sends it to the DeepSeek TEE via the 0G broker. The TEE processes the inference in an isolated environment, returns a **signed response** (signal + confidence + attestation hash), and that signature is recorded on-chain via `VerifierEngine.attest()`.

**Why this matters for the product:** The TEE signature is the trust anchor. Without it, an AI trading system's signals could be cherry-picked or fabricated. With 0G Compute TEE, the inference is cryptographically provable — the `attestationHash` stored on-chain can only have been generated by an authentic run of the model inside the TEE.

**SDK integration** (`agent/src/attester.ts`):
```typescript
const broker = await createZGComputeNetworkBroker(wallet);
const { endpoint, model } = await broker.inference.requestProcessor
  .getServiceMetadata(DEEPSEEK_PROVIDER);
const headers = await broker.inference.requestProcessor
  .getRequestHeaders(DEEPSEEK_PROVIDER, query);
// encrypted query sent to TEE; response includes attestation hash
const isValid = await broker.inference.responseProcessor
  .processResponse(DEEPSEEK_PROVIDER, responseId, content);
```

### ✅ Component 3: 0G Storage (Decision Archive)

**What it is:** Decentralized, content-addressed storage network used as the permanent archive layer for every AI decision PROVUS makes.

**How PROVUS uses it:** Every 50 iterations, the agent serializes the batch of decisions (signal, confidence, attestation hash, timestamp, market context) into a JSON blob, uploads it to 0G Storage, then writes the resulting CID + Merkle root to the `ArchiveRegistry` contract on 0G Chain. This gives PROVUS a two-tier proof: the lightweight on-chain attestation for real-time verification, and the full decision payload preserved in 0G Storage for forensic replay.

**Why this matters for the product:** On-chain calldata is expensive and bandwidth-limited, but auditors and regulators need the full decision context — the prompt, the TEE response, the volatility regime — to verify a strategy after the fact. 0G Storage gives PROVUS a path to keep every byte of every decision permanently retrievable while only paying for a Merkle root on chain.

**Contract:** [`ArchiveRegistry` — `0x8fa2c5ae17E2D170C54fC3CA34148B0Ad503d8DB`](https://chainscan.0g.ai/address/0x8fa2c5ae17E2D170C54fC3CA34148B0Ad503d8DB)

## The Problem

AI trading systems are inherently unverifiable. Anyone can claim a 79% win rate — there is no standard for proving that AI decisions were made *before* the market moved, not reverse-engineered and backdated.

- Off-chain logs can be forged, altered, or predated after the fact
- Centralized attestation creates a single point of failure
- Regulators cannot audit real-time inference logic — it is a black box
- Batch attestations hide exact decision timing

Result: **$2.3B/year** in algorithmic trading fraud (SEC 2024 report). No cryptographic standard exists for proving AI decision integrity in real time.

## The Solution: How PROVUS Works

Every 15 seconds, the PROVUS agent runs this cycle autonomously:

| Step | Action | Technology |
|---|---|---|
| 1 | Fetch ETH/USDT OHLCV (144 × 5-min candles) | Binance REST API |
| 2 | Calculate Yang-Zhang realized volatility (12-hour window) | Agent — `volatility.ts` |
| 3 | Classify regime: LOW / MEDIUM / HIGH / EXTREME | Agent — `volatility.ts` |
| 4 | Encrypt market context query, send to TEE | **0G Compute** — `attester.ts` |
| 5 | Receive signed inference: `signal + confidence + teeProofHash` | **0G Compute** — DeepSeek V3.1 |
| 6 | Record volatility on-chain: `recordVolatility()` | **0G Chain** — VerifierEngine |
| 7 | Attest AI decision on-chain: `attest()` | **0G Chain** — VerifierEngine |
| 8 | Update agent ELO reputation based on signal accuracy | **0G Chain** — ReputationEngine |
| 9 | Broadcast updated proof state to frontend | Agent HTTP server |

**Key invariant:** Steps 4–5 (TEE inference) always precede steps 6–7 (on-chain attestation). The block timestamp on each `attest()` transaction is the cryptographic proof that the signal existed before the next market move.

## Smart Contracts (Solidity 0.8.24 · Deployed on 0G Chain)

### VerifierEngine — `0x911E87629756F34190DF34162806f00b35521FD0`
The attestation hub. Called twice per 15-second cycle. Stores every AI decision as an immutable on-chain record with TEE proof hash and block timestamp.

- `recordVolatility(strategyId, taskId, volBps, regime)` — seals the volatility context
- `attest(strategyId, taskId, attestationHash, storageRoot, signal, confidence, isValid)` — seals the AI decision
- Emits `DecisionVerified` events (subscribable by other protocols)

### ReputationEngine — `0x57C7f2F3051928E2cc7C871Bac590bF1d4BF4c8e`
ELO scoring system. Updates after every verified signal. K-factor = 32 (chess standard). Current PROVUS ELO: **847**.

- `recordStrategy(strategyId, signal, confidence, wasCorrect)` — updates ELO
- `getAgentReputation(strategyId)` → `(eloScore, decisionsCount, winRate)` — queryable by external protocols

### StrategyRegistry — `0x87E3D9fcfA4eff229A65d045A7C741E49b581187`
ERC-721 contract. Each trading strategy is minted as a non-fungible token. Transferable, composable, enumerable.

### StrategyVault — `0x2B9366b7fea6a1C6279edbC7B87CCB91CdCc1014`
Capital management + trade execution layer. Executes hedges based on `attest()` signals with slippage protection (±5% delta tolerance).

**Security:** Audited via ChainGPT AI (May 2026). No critical or high-severity findings. See [AUDIT.md](./AUDIT.md) for the full report.

## Local Deployment (Reproduction Steps)

### Prerequisites

- Node.js v18+ (v24 recommended)
- A 0G Chain wallet with OG tokens for gas ([faucet: https://faucet.0g.ai](https://faucet.0g.ai))
- A 0G Compute ledger funded with OG tokens (for TEE inference)
- Git

### 1. Clone & Install

```bash
git clone https://github.com/Gideon145/provus-protocol.git
cd provus-protocol
npm install
cd contracts && npm install && cd ..
cd agent && npm install && cd ..
cd frontend && npm install && cd ..
```

### 2. Configure Environment

```bash
cp .env.example .env
```

Edit `.env` with your values:

```env
# 0G Chain
ZG_PRIVATE_KEY=0x...                   # Your 0G wallet private key
ZG_RPC_URL=https://evmrpc.0g.ai        # 0G Chain mainnet RPC
ZG_CHAIN_ID=16661

# Deployed contract addresses (already live — use these)
VERIFIER_ENGINE=0x911E87629756F34190DF34162806f00b35521FD0
REPUTATION_ENGINE=0x57C7f2F3051928E2cc7C871Bac590bF1d4BF4c8e

STRATEGY_ID=1

DEEPSEEK_PROVIDER=0xd9966e13a6026Fcca4b13E7ff95c94DE268C471C
```

## Composability

PROVUS contracts are designed as open primitives. External protocols can:

- **Subscribe** to `DecisionVerified` events on VerifierEngine for real-time AI signals
- **Query** `getAgentReputation(strategyId)` on ReputationEngine to gate access by ELO
- **Route** capital through StrategyVault to verified, high-ELO agents
- **Enumerate** active strategies on StrategyRegistry (ERC-721 enumerable)

Example integration:
```solidity
IReputationEngine rep = IReputationEngine(0x57C7f2F3051928E2cc7C871Bac590bF1d4BF4c8e);
(uint256 elo,,) = rep.getAgentReputation(1);
require(elo > 800, "Agent ELO too low");
// execute auto-rebalance using PROVUS signal
```
